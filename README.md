# Resender TG Bot

Бот пересылает сообщения (текст, фото, PDF, DOCX и любые вложения) из топиков
супергруппы-источника **«Негр чит work group»** в нужные топики
супергруппы-назначения **«Не горчит складского учёта»**.

Маршрутизация — по паре **источник (чат + топик) + хэштег**.
Способ пересылки — **forward** (сохраняется пометка «Переслано от …»).

Бот работает в двух режимах:

| Режим      | Где              | Файл            | Назначение                          |
|------------|------------------|-----------------|-------------------------------------|
| **polling**| локально         | `bot.py`        | отладка, получение ID топиков (`/id`) |
| **webhook**| Vercel (serverless) | `api/index.py` | боевой режим                        |

## Что куда пересылается

Настраивается в `config/routes.yaml`. Числовые ID топиков там — **заглушки**,
их нужно заменить реальными (см. шаг 4).

| Источник (топик)              | Назначение (топик)                          |
|-------------------------------|---------------------------------------------|
| маркова поставки накладные    | Маркова поставки перемещения и списания     |
| маркова списания              | Маркова поставки перемещения и списания     |
| желток сан поставки накладные | Желток сан поставки                         |
| желток сан списания           | Желток сан списания                         |

## Локальная настройка (делается ОДИН раз, до деплоя)

### 1. Установка

```bash
python3 -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Токен

Файл `.env` уже создан с токеном. **Важно:** в @BotFather для бота должен быть
отключён режим приватности, иначе он не видит обычные сообщения в группах:
`/setprivacy` → выберите бота → **Disable**.

### 3. Узнать ID чатов и топиков

Запустите бота локально:

```bash
python bot.py
```

В каждом нужном топике обеих супергрупп напишите `/id` — бот ответит:

```
chat_id: -1001234567890
thread_id (топик): 11
```

### 4. Заполнить config/routes.yaml

Подставьте реальные числа вместо заглушек:
`source_chat_id`, `dest_chat_id`, `source_thread_id`, `dest_thread_id`.
Если топик — «General» (общий), бот покажет `thread_id: нет` — тогда оставьте
`source_thread_id: null`.

Фильтр по хэштегам (необязательно): пустой список `[]` = пересылать все сообщения
из топика; иначе перечислите хэштеги в `match_hashtags`.

### 5. Проверить локально

Снова `python bot.py`, отправьте тестовое сообщение/фото/PDF в топик-источник —
оно должно появиться в нужном топике назначения.

## Версионирование: GitHub → Vercel

### 6. Git и пуш на GitHub

Репозиторий уже инициализирован, первый коммит создан. Выполните в терминале
**на своём компьютере** (не в облаке):

```bash
cd "путь/к/Resender TG Bot NG"

# Если git ругается на .lock-файлы (наследие первой инициализации):
rm -f .git/*.lock .git/objects/*.lock

git remote add origin https://github.com/<ваш_аккаунт>/<репозиторий>.git
git branch -M main
git push -u origin main
```

> `.env` в `.gitignore` — токен **не** попадёт в репозиторий. Это правильно.

### 7. Деплой на Vercel

1. Зайдите на [vercel.com](https://vercel.com) → **Add New… → Project** → импортируйте
   репозиторий с GitHub. Vercel сам определит Python-функцию в `api/`.
2. В **Project → Settings → Environment Variables** добавьте:
   - `BOT_TOKEN` — токен бота;
   - `WEBHOOK_SECRET` — любая случайная строка (например, из `openssl rand -hex 16`);
   - `ROUTES_FILE` — `config/routes.yaml` (необязательно).
3. Нажмите **Deploy**. Получите адрес вида `https://<project>.vercel.app`.

Дальнейшие `git push` в `main` будут автоматически деплоиться на Vercel — это и
есть версионирование.

### 8. Привязать webhook к Telegram

**Простой способ (рекомендуется):** после деплоя один раз откройте в браузере
`https://<project>.vercel.app/api/setup` — эндпоинт сам зарегистрирует webhook
(использует `BOT_TOKEN` и `WEBHOOK_SECRET` из переменных окружения Vercel).
В ответе должно прийти `"ok": true`.

**Альтернатива — вручную локально**, с тем же `WEBHOOK_SECRET` в `.env`:

```bash
python set_webhook.py set https://<project>.vercel.app/api/index
python set_webhook.py info     # проверить статус
python set_webhook.py delete   # снять webhook (например, чтобы снова отлаживать локально)
```

Проверка, что функция жива: откройте `https://<project>.vercel.app/api/index`
в браузере — должна вернуться строка «webhook endpoint is live».

> ⚠️ В одно время активен только один режим: либо webhook (Vercel), либо polling
> (локально). Перед локальной отладкой снимайте webhook (`delete`), после —
> ставьте обратно (`set`).


## Важные заметки

- **Альбомы.** На Vercel (serverless) нет общей памяти между запросами, поэтому
  альбом из нескольких вложений пересылается **по одному сообщению**, а не единым
  блоком. Все файлы доходят. Если используете фильтр по хэштегам — учтите, что
  подпись обычно есть только у первой части альбома.
- Поддерживаются любые вложения: фото, документы (PDF, DOCX, XLSX…), видео и т.д.
- Сообщения, не подходящие ни под один маршрут, игнорируются.
- Токен, ранее отправленный в переписке, стоит перевыпустить в @BotFather
  (`/revoke`) и обновить в `.env` и в переменных окружения Vercel.
